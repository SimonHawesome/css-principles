The Big 3 Browser Rendering Terms

Layout > Paint > Composite

Layout > Paint > Composite

  • Calculation of how much space an element takes on the screen.
    Geometric properties: width, height, display, font-size, top, left...etc
  • When modified, the browser will have to "reflow" the page
  • "Reflow" means also repainting and compositing the affected areas

Layout > Paint > Composite

  • Fills visual elements (color, background, border-colors)
  • Can skip layout, but causes a recomposition
  • Most expensive $$$

Layout > Paint > Composite

  • Draws layers in the intended order i.e. overlapping elements
  • Any style modification requires a recomposition
  • ..but z-index still needs a repaint

Let's talk performance

...baby!

Browser reads selectors from right to left

						
							.container ul li a { }
  • Match every 'a' tag
  • Find every 'a' contained in 'li'
  • Use previous matches and find ones contained in 'ul'
  • Finally, filter down the matches contained in '.container'
  • It's expensive AF, bud.

Skip all of that mombo jumbo by adding a class to the 'a' tag

						
							

							.container-link { }

Some of the most expensive CSS properties

  • border-radius
  • box-shadow
  • filter
  • :nth-child
  • position: fixed

...but you can still use them!

Just be wary of how many times the property is rendered i.e. hundreds of times vs. 20 times
